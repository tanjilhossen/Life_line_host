=========================================================
      PROJECT STATUS REPORT: LIFELINE (BLOOD & HOSPITAL)
=========================================================
This document outlines the current state, architecture, and completed features of the "LifeLine" project. It is designed to provide full context to an AI assistant for future development.

1. PROJECT OVERVIEW
---------------------------------------------------------
LifeLine is a comprehensive web application designed to connect blood donors, hospitals, and people in need of emergency blood. It features separate dashboards for Donors, Hospitals, and a Master Admin, with advanced location tracking, real-time emergency blood request handling, and automated WhatsApp notifications.

2. TECHNOLOGY STACK
---------------------------------------------------------
- Backend: Node.js, Express.js
- Database: MySQL (Database Name: lifeline_db)
- Frontend: HTML5, CSS3 (Vanilla), Vanilla JavaScript
- Key APIs/Integrations: 
  * Google Maps API & Places API (Location mapping & search)
  * whatsapp-web.js (Automated WhatsApp messaging & alerts)
  * qrcode-terminal (WhatsApp auth)

3. DATABASE ARCHITECTURE (Ready Tables)
---------------------------------------------------------
1. hospitals: Stores hospital profiles, bed capacities (ICU, emergency), precise location (lat/long), and login credentials.
2. donors: Stores donor profiles, blood groups, last donation date, physical location (lat/long), donation count, and status.
3. applications: A staging table for new donor/hospital registrations. Awaiting approval or rejection from the Master Admin.
4. blood_requests: Stores emergency blood requests (blood group, location, hospital name, needed time, patient disease).
5. donor_requests: A relational table tracking donor responses (Pending, Accepted, Rejected) to specific blood requests.

4. COMPLETED MODULES & FEATURES
---------------------------------------------------------

[A] AUTHENTICATION & REGISTRATION
- Multi-role registration (Donors & Hospitals).
- Application approval system (Users cannot log in until the Master Admin approves their application).
- Secure login portal.

[B] DONOR DASHBOARD (donor-dashboard.html)
- Profile Management: Donors can update their contact info, health notes, and last donation date.
- Location Tracking: Interactive Google Maps integration allowing donors to pin their exact physical location in Bangladesh.
- Emergency Blood Request Feed: 
  * Pending requests are sorted and pinned at the top.
  * Implemented a 5-minute interactive countdown timer for pending requests. Action buttons expire after 5 mins.
  * Donors can "Accept" or "Reject" requests.
  * Recently accepted requests (within a 5-hour window) are displayed below pending requests.
  * System tracks "No Response" if the donor ignores the request.
- Automated Communication: Upon accepting a request, donors receive automated WhatsApp messages with the requester's direct contact details.

[C] HOSPITAL DASHBOARD (hospital-dashboard.html)
- Profile & Resource Management: Hospitals can update their available ICU and emergency bed counts in real-time.
- Location Tracking: Interactive Google Maps integration to set exact hospital coordinates.

[D] MASTER ADMIN DASHBOARD (master-dashboard.html)
- Application Management: Admin can review, approve, or reject incoming registration applications from hospitals and donors.

[E] EMERGENCY BLOOD REQUEST WORKFLOW
- Users can submit an emergency blood request with critical details (disease, time, hospital, location).
- The system matches requests using MySQL LIKE queries based on district-upazila location mapping.
- Requests are instantly routed to matching donors' dashboards.

5. KNOWN INTEGRATION DETAILS
---------------------------------------------------------
- Google Maps: Replaced Mapbox. Locked to Bangladesh territory. Uses Places API for location autocomplete and Maps API for visual selection.
- WhatsApp Bot: Runs via whatsapp-web.js on the Node server. Authentication session is cached locally.
